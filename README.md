## 👋 はじめまして、rx-tomo です

社内向けの業務システムや BtoB SaaS を中心に、設計・実装・運用の各フェーズで開発に取り組んでいます。
特に最近は、エンタープライズ向けSaaSプロダクトの開発に注力しながら、E2Eテスト自動化や要件定義の体系化にも力を入れています。

> ⚠️ 実際のコードは機密性の高い社内プロジェクトが多いため公開できませんが、技術的な取り組みや関心領域は積極的に発信していきたいと考えています。

---

### 🧠 関心のあるテーマ・取り組んでいる技術

- **エンタープライズSaaS開発**
  Next.js (App Router) + TypeScript + Supabase によるマルチテナントSaaS構築、RLS によるテナント分離
- **E2Eテスト自動化**
  Playwright によるジャーニーテスト、RBAC検証、マルチユーザー承認フローのテスト自動化
- **要件定義の体系化**
  EARS (Easy Approach to Requirements Syntax) 形式による要件定義、約280件の要件を体系的に整備
- **データパイプライン**
  dbt + Prefect によるETL構築、PostgreSQL連携、オープンデータの正規化・分析基盤
- **GIS / 3D可視化**
  MapLibre GL JS + TerraDraw による地図アプリ、3D建物表現と日影シミュレーション
- **GitOps / DX 自動化**
  GitHub Actions × Claude Code による開発フロー自動化、MCP (Model Context Protocol) 活用
